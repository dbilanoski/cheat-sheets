# Python Project Structure
Optimal structure for the project which will have:
* modules
* testing
* version control
* packaging and deployment

can follow this structure:

```text
my-app/
├─ modules/
│  ├─ __init__.py
├─ tests/
│  ├─ test_cases.py
├─ main.py
├─ requrements.txt
├─ .gitignore
├─ README.md

```
